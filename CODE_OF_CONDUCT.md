# Code of conduct

## Our commitment

Edgaze is a **closed-source, proprietary platform** developed and maintained by authorized team members only. We are committed to fostering a professional, inclusive, and collaborative environment for all contributors.

---

## Our standards

### Positive behaviors

Contributors are expected to:

- **Be professional** — Maintain high standards in all interactions
- **Be respectful** — Value different perspectives and experiences
- **Be collaborative** — Work together towards shared goals
- **Be constructive** — Provide helpful, actionable feedback
- **Be accountable** — Take responsibility for your work and decisions
- **Be security-conscious** — Protect proprietary information and follow security practices

### Unacceptable behaviors

The following behaviors are not tolerated:

- Harassment, discrimination, or intimidation of any kind
- Inappropriate comments or personal attacks
- Sharing proprietary code or information without authorization
- Deliberately introducing bugs or security vulnerabilities
- Violating confidentiality agreements
- Any behavior that creates an unsafe or unprofessional environment

---

## Responsibilities

### Contributors

All contributors are responsible for:

- Following this code of conduct
- Maintaining confidentiality of proprietary information
- Reporting concerns promptly and professionally
- Adhering to security and development standards

### Maintainers

Project maintainers are responsible for:

- Enforcing this code of conduct fairly and consistently
- Addressing concerns in a timely manner
- Maintaining a safe and professional environment
- Leading by example

---

## Scope

This code of conduct applies to:

- All project repositories and code (internal only)
- Issue trackers and pull requests (internal)
- Team communications (Slack, email, meetings)
- Internal project discussions and events
- Any other project-related spaces

**Note:** As a closed-source platform, all project activities are internal and confidential.

---

## Enforcement

### Reporting

If you experience or witness unacceptable behavior:

1. **Contact project maintainers** through designated channels
2. **Provide specific details** about the incident
3. **Include any relevant evidence** (screenshots, messages, etc.)

All reports will be handled confidentially and investigated promptly.

### Consequences

Violations may result in:

- **Warning** — Formal notice of the violation
- **Temporary suspension** — Temporary loss of repository access
- **Permanent removal** — Termination of access and involvement
- **Legal action** — In cases of severe violations (e.g., data theft)

Consequences depend on the severity and frequency of violations.

---

## Confidentiality and security

### Proprietary information

All contributors must:

- Treat all code, documentation, and discussions as confidential
- Never share or discuss proprietary information publicly
- Use secure methods for communication and data handling
- Report any potential data breaches immediately

### Security practices

- Follow security guidelines in [SECURITY.md](SECURITY.md)
- Never commit secrets, API keys, or credentials
- Report vulnerabilities privately
- Use secure authentication and access controls

---

## Professional standards

### Code quality

- Write clean, maintainable code
- Follow established coding standards
- Document complex logic
- Test your changes thoroughly

### Collaboration

- Provide constructive code reviews
- Be receptive to feedback
- Communicate clearly and respectfully
- Help onboard new team members

### Communication

- Be clear and concise
- Assume good intent
- Respond to questions and requests promptly
- Use appropriate channels for different types of communication

---

## Intellectual property

All contributors acknowledge that:

- All work produced belongs to Edgaze
- Contributions are made under the terms of your employment or contributor agreement
- You will not reuse proprietary code elsewhere without authorization
- You respect the intellectual property rights of others

## Questions and clarifications

If you have questions about this code of conduct:

- Contact project maintainers
- Discuss during team meetings
- Refer to [SUPPORT.md](SUPPORT.md) for contact information

This code of conduct may be updated periodically. Contributors will be notified of significant changes. Continued participation implies acceptance of the current version.
